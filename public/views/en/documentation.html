Getting started

This API is designed to perform numerical conversions between different data types. For example, converting from 10 decimal to 1010 binary or from 3.5 decimal to three point fivetext.

All requests must be sent using the POST method to the following URL:

https://www.numberconverterapi.kodexiv.com/api/converter

Make your first conversion request now.

First request

Below is a basic example of how to perform a conversion using the API.

All requests must be sent using the POST method to the following URL:

https://www.numberconverterapi.kodexiv.com/api/converter

Conversion requests require only two sections: from, which specifies the value to convert, and to, which indicates the expected response.

The from section must include two fields: the value to be converted and its type. The to section only requires the type of the expected result.

For example, to convert the value 10 decimal to binary, you must send the following request: 

{
  "from": {
    "value": "10",
    "type": "decimal"
  },
  "to": {
    "type": "binary"
  }
}

The result of the conversion will be:

{
  "data": "1010"
}

Request structure

The structure of the requests consists of only two elements: from, which indicates the value to be converted, and to, which indicates the expected response.

From

This object contains the initial value of the conversion. It must include the value to be converted and its type. Optionally, it may also include formats that describe the structure of the input value.

To

This object defines the data type expected as the result. Optionally, it may include formats that specify the desired structure of the output.

Structure

Based on the above, the structure of a request is as follows: 

{
  "from": {
    "value": "",
    "type": "",
    "format": {}
  },
  "to": {
    "type": "",
    "format": {}
  }
}

If additional elements are sent, they will be ignored by the API.

Response structure

The API can return three types of responses, depending on the values sent in the request.

Ideal response

This response is returned when no error or warning occurs during the conversion. The response consists only of the data object, which contains the final result.

Request

{
  "from": {
    "value": "10",
    "type": "decimal"
  },
  "to": {
    "type": "binary"
  }
}

Response

{
  "data": "1010"
}

Response with warnings

This response is returned when a non-critical condition occurs during the conversion. The response includes the warnings object, and data may contain the final conversion value or not, depending on the situation.

Request

{
  "from": {
    "value": "A",
    "type": "decimal"
  },
  "to": {
    "type": "binary"
  }
}

Response

{
  "warnings": {
    "validation": "`A` is not a valid decimal number."
  },
  "data": null
}

Response with errors

This response is returned when the conversion cannot be performed, usually due to an invalid value or a missing required element. In this case, only the errors object is returned.

Request

{
  "from": {
    "value": "10"
  },
  "to": {
    "type": "binary"
  }
}

Response

{
  "errors": {
    "from.type": "The `from.type` element is required."
  }
}

Error structure

Errors are objects returned in the response of a conversion. They are generated when the request contains an incorrect value or when a required element is missing.

Structure

The errors object contains a set of key-value pairs, where the key identifies the element that produced the error and the value is the corresponding error message.

Request

{
  "from": {
    "type": null
  },
  "to": {
    "type": "base10"
  }
}

Response

{
  "errors": {
    "from.type": "The `from.type` element cannot be null.",
    "from.value": "The `from.value` element is required.",
    "to.type": "'base10' is not a valid `to.type` value. Accepted values are: 'hexadecimal', 'decimal', 'octal', 'binary', 'base62', 'base64', 'text'."
  }
}

Warning structure

Warnings are objects returned in the response of a conversion. These are generated when, during the conversion, any of the following events occurs:

  1. An invalid value is sent for the selected type. In this case, the data will always return null.
  2. The value to convert is sent in numeric form (the string form is preferable to maintain precision with very large numbers).
  3. A separation is specified in the from object that is not used by the value sent.
  4. A conversion with a decimal part cannot produce an exact result. In this case, the decimals in the response are truncated to 25 characters.
  5. A conversion is performed where to.type is set to text and the number sent for that conversion is too large. In this case, the data object will always return NTL (number too large).
  6. A conversion is performed where from.type is set to text and the value sent is not a valid number. In this case, the data object will always return NaN (not a number).

Structure

The warnings object contains a set of key-value pairs, where the key identifies the element that produced the warning and the value is the corresponding message. It is important to note that warnings 4, 5, and 6 use data as their key, since they are considered issues related to the result of the conversion and not to the from element sent.

Example

Request

{
  "from": {
    "value": 10.9,
    "type": "decimal",
    "format": {
      "separation": "comma"
    }
  },
  "to": {
    "type": "binary"
  }
}

Response

{
  "warnings": {
    "from.value": "To maintain precision, always try to send `from.value` in string format, not number format: '10.9' instead of 10.9.",
    "from.format.separation": "The value sent did not have the specified separation.",
    "data": "The result had too many decimals. These were truncated to only twenty-five characters."
  },
  "data": "1010.11100110011001100110011"
}
